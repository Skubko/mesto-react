# mesto-react
Всем привет! Немного полезных материалов по текущей практической работе.
По заданию вам требуется переписать ранее написанное приложение на React. Задача звучит страшнее, чем есть на самом деле, и она точно проще чем проектная работа 9. В целом описание очень адекватно и проблем возникнуть не должно. Но есть несколько моментов на которые надо обратить внимание:
В результате работы у вас должно получится как минимум 7 компонентов, которые будут лежать в папке components - App, Card, Footer, Header, ImagePopup, Main, PopupWithForm. Также у вас будет основной компонент необходимый для рендеринга готового приложения в DOM-дереве. Его можно расположить в основном js файле (index.js)
Компонент Card - это репрезентация вашего класса Card. По сути внутри него в рамках данной проектной работы нужны 2 пропса: (1) функция-колбэк для открытия попапа которая придет “сверху“, из компонента Main, где карточка отрисовывается; (2) сами данные карточки, которые вы передаете из цикла при начальной отрисовки. Соответсвенно у вас внутри класса Card будет сейчас только разметка + вызов функции при клике на картинку
C компонентами Footer и Header проблем быть не должно. Это просто разметка с подключением картинок через import. Будет круто если вы посмотрите в сторону того чтобы не “хардкодить” 2020 год у себя в компоненте, а высчитывать его при рендеринге внутри разметки через объект Date. Иначе каждую новогоднюю ночь вам придется делать коммит с обновлением)
У вас будет всего два компонента под попапа (вместо, как минимум четырех, что были раньше) - ImagePopup и PopupWithForm. ImagePopup будет принимать только два props (ссылка на функцию-колбэк закрытия и данные карточки) и использовать их в разметке. Чтобы не прописывать в разметке undefined в тех случаях когда никакая карточка не выбрана - посмотрите в сторону использования тернарных операторов. Показывать попап в разметке или нет можно тоже через модификатор (добавлять модификатор, если выбранная карточка есть). PopupWithForm будет получать уже больше props: (1) заголовок формы, (2) children с самой формой, (3) заголовок для кнопки под формой, (4) булево значение isOpen чтобы понимать закрываем попап или открываем, (5) в случае необходимости name для идентификации формы + еще какие-то props если они нужны
Внутри файла App.js вы собираете все компоненты воедино и создаете по крайней мере 4 стейта через хук - для состояния трех попапов (один который нужен для удаления пока всегда закрыт) + один стейт чтобы держать инфу о текущей карточке которую надо открыть в попапе после нажатия. Обратите внимание что по заданию должна быть одна общая функция для закрытия всех попапов - closeAllPopups. Можете в ней прописать логику чтобы все нужные стейты переводились в состояние false
